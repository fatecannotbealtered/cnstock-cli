# Code of Conduct

## Our Pledge

We aim to keep cnstock-cli a practical, respectful project for users and contributors. Contributions should focus on technical clarity, reproducible evidence, and responsible use of public market-data endpoints.

## Expected Behavior

- Be respectful and direct.
- Discuss technical disagreements with concrete examples, logs, tests, or code references.
- Respect data-source limitations, licensing boundaries, and security disclosure rules.
- Avoid harassment, personal attacks, discriminatory language, or repeated off-topic disruption.

## Unacceptable Behavior

- Publishing private vulnerability details before maintainers have had a reasonable chance to respond.
- Encouraging abuse of upstream endpoints, high-frequency scraping, automated trading, or compliance-sensitive use.
- Sharing secrets, tokens, private credentials, or non-public market data in issues, PRs, or discussions.

## Enforcement

Maintainers may edit, hide, or remove comments and may block contributors who violate this code. Security issues should be reported through the private process in [SECURITY.md](SECURITY.md), not through public issues.
